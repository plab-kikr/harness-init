# Review — 코드 리뷰

## 목적

변경된 코드가 프로젝트의 아키텍처 규칙과 품질 기준을 충족하는지 검토합니다.

## 실행 절차

1. **변경 범위 파악**
   - 변경된 파일 목록 확인 (`git diff --name-only`)
   - 변경의 목적 확인

2. **CLAUDE.md 기준 검토**
   - 아키텍처 규칙 위반 여부
   - "절대 하지 말 것" 항목 위반 여부
   - 코드 변경 체크리스트 항목 확인

3. **코드 품질 검토**
   - 중복 코드 없는지
   - 불필요한 복잡도 없는지
   - 에러 처리 적절한지

4. **보안 검토**
   - 환경변수 하드코딩 없는지
   - 사용자 입력 검증 여부
   - 인증/권한 처리 적절한지

5. **빌드/린트 실행**
   - CLAUDE.md의 개발 명령어로 검증

## 출력 형식

```
## 리뷰 결과

**통과:** [문제 없는 항목]
**수정 필요:** [문제 항목과 이유]
**제안:** [선택적 개선 사항]
```
