# Debug — 체계적 디버깅

## 목적

증상에서 시작해 근본 원인까지 추적합니다.
추측으로 코드를 변경하지 않고, 증거를 먼저 수집합니다.

## 실행 절차

### 1단계: 증상 수집
- 정확한 오류 메시지 확인
- 재현 조건 파악 (항상 발생? 특정 상황에서만?)
- 최근 변경 사항 확인 (`git log`, `git diff`)

### 2단계: 가설 수립
- 오류 발생 가능한 원인 3가지 이상 나열
- 가능성 높은 순으로 정렬

### 3단계: 증거 수집
- 로그, 스택 트레이스 분석
- 관련 코드 경로 추적
- 각 가설을 검증하는 최소한의 테스트

### 4단계: 근본 원인 확인
- 재현 가능한 최소 케이스 만들기
- 원인이 확실해질 때까지 추측으로 수정하지 않음

### 5단계: 수정 및 검증
- 최소 범위로 수정
- 동일 증상이 재발하지 않음을 확인
- 관련 엣지 케이스 확인

## 금지 사항

- 원인 파악 전 코드 변경 금지
- "아마 이게 문제일 것 같다"는 추측으로 수정 금지
- 같은 수정을 반복 시도 금지 (다른 접근 방법 탐색)
