## 확인 우선 — 추측 금지

- **diff 범위 질문**이 나오면 반드시 `git diff --name-only` 실행 후 답변. "건드리지 않은 파일"이라는 추측 금지.
- **파일 존재 여부 확인** 없이 "이 파일에는 없다"고 단정 금지. grep/find 실행 후 답변.
- **타입/ID 의미** 불명확 시 코드에서 직접 추적한 뒤 답변. 이름으로 추측 금지.

---

## 외부 도구 / AI 리뷰 검증

- **Gemini/AI 코드리뷰**는 제안 코드가 실제 코드 구조(URL 패턴, 타입 정의, 레이어 구조)와 맞는지 대조 확인 후 적용. 우선순위(HIGH/MEDIUM)와 무관하게 검증 먼저.
- **제안이 수정하려는 버그를 재발시키는 경우** SKIP하고 사유를 PR 코멘트에 명시.
- KISS/YAGNI 기준 판단: 단일 사용 코드에 추상화 추가, 존재하지 않는 미래 케이스 대비 → SKIP.

---

## 타입 안전성 (TypeScript)

- `useState<string>`에 `string | undefined` 반환 함수 결과 직접 전달 금지 → `?? ''` fallback 필수.
- 함수 반환 타입과 state setter 타입은 코드 작성 시점에 맞춰서 확인.

---

## 테스트 변경 전 기존 기대값 확인

- 함수 정책(마스킹 룰, 계산식 등)을 변경하면 관련 테스트 파일을 전부 grep해서 기존 기대값 목록화 후 일괄 업데이트.
- 테스트 파일이 두 곳 이상에 흩어져 있을 수 있음. `grep -r "기대값" tests/` 로 전체 탐색.
